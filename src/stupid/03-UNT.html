Untestability

Testear no debería ser complicado.

Unit testing es importante, si no testeas tu código estas entregando un producto incompleto.

Aún así hay gente que no cubre apropiadamente su código con tests.

¿Por qué?

La mayoría de los casos el código es difícil de testear, ¿qué lo hace difícil de testear? Principalmente el punto anterior, tight coupling.

Unit testing refiere a testear unidades de código, generalmente clases, pero ¿cómo puedes testear clases individuales si están altamente acopladas?

Cuando no escribes tests porque “no tenes tiempo” la causa real probablemente es que tu código sea malo, si tu código es de calidad entonces puedes testearlo de forma rápida.
